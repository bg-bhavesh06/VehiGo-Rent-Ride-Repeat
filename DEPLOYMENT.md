# Render Deployment

This project is configured as a single Render Web Service. Render builds the React client, then starts the Express server. Express serves both the API routes and the compiled client from `client/dist`.

## Deploy with Blueprint

1. Push this repository to GitHub.
2. In Render, create a new Blueprint and select this repo.
3. Render will detect `render.yaml` at the repo root.
4. Fill in the prompted environment variables.
5. Deploy.

## Required Environment Variables

- `MONGO_URI`
- `CLOUDINARY_CLOUD_NAME`
- `CLOUDINARY_API_KEY`
- `CLOUDINARY_API_SECRET`
- `RAZORPAY_KEY_ID`
- `RAZORPAY_KEY_SECRET`
- `VITE_RAZORPAY_KEY_ID`
- `GROQ_API_KEY`
- `EMAIL_USER`
- `EMAIL_PASS`

`JWT_SECRET` is generated by Render from `render.yaml`. Set `CLIENT_ORIGIN` only if you later host the client separately or want to restrict browser origins to specific domains.

## Commands Used by Render

- Build: `npm ci --prefix server && npm ci --prefix client && npm run build --prefix client`
- Start: `npm start --prefix server`
